-   **База:** SQLite + Alembic. Поля `is_debt`, `is_goal` добавлены.
-   **Brain:** v2.1 — Pydantic с fallback-логикой для пропущенных полей.

## 📅 2026-04-17: Фикс стабильности и визуала

### ✅ Что сделано:
1.  **Анимация ожидания**: 3 сообщения с "бегущими" точками, старые удаляются при ответе.
2.  **Формат чисел**: Функция `fmt_money()` — `216750` → `216.8к` для удобства чтения.
3.  **Формула "Итого"**: Считает только `нераспределённые + остаток расходных + накоплено в целях` (долги не суммируются).
4.  **Пост-валидация**: Код обрезает `current_amount > target_amount`, даже если LLM ошибся.
5.  **Визуал**: 🚨 вместо ⚠️, переносы в фин-сводке, убрана лишняя фраза про конверты.
6.  **Fallback для JSON**: Если LLM забыл поле `thoughts`/`intent` — код подставит дефолт, бот не упадёт.

### ❌ Блокеры:
-   LLM иногда пропускает обязательные поля в JSON, несмотря на примеры в промпте. Решение: fallback-логика в коде.
-   Стоимость ~0.60₽/запрос — приемлемо для тестов, но для продакшена нужно оптимизировать промпт дальше.

### 💡 Инсайты:
-   **Анимация важна**: Пользователь терпит 50 сек ожидания, если видит прогресс ("Думаю..." → "Почти готово").
-   **Пост-валидация спасает**: Даже умная модель ошибается в арифметике. Код должен быть "последней линией обороны".
